# Selection Guide

Choose one primary recipe. Add a secondary recipe only when the visual must do
two jobs, such as a technical diagram with a code-review warning overlay.

## Article Illustration

Use for essays, blog posts, newsletters, opinion pieces, Notion docs, and
conceptual explanations.

Good visual jobs:

- Make an abstract claim memorable.
- Show a cognitive turn or contrast.
- Visualize a metaphor.
- Break a long text with one idea per image.

Avoid when the user needs exact system structure or implementation details.

## Technical Diagram

Use for systems, RPC, database, state machines, cloud/server design, APIs,
pipelines, deployment, storage, observability, and architecture docs.

Good visual jobs:

- Show boundaries and ownership.
- Show data/control flow.
- Show state transitions.
- Explain old vs new system behavior.

Avoid when the request is primarily editorial or emotional.

## Code Review Visual

Use for review reports, bug explanations, regression paths, failure modes,
coverage gaps, risk maps, and before/after behavior.

Good visual jobs:

- Show how a bug propagates.
- Show the missing guard or test.
- Compare current and expected behavior.
- Make severity and blast radius visible.

Avoid when the user only needs prose findings.

## Product Explainer

Use as an extension of article illustration or technical diagram for README,
landing page, docs intro, and feature walkthrough visuals.

Prefer concise product-state visuals over marketing hero art.

## Workflow Map

Use as an extension of technical diagram for agent workflows, data pipelines,
review loops, content pipelines, or task lifecycles.

Focus on stages, handoffs, and feedback loops.

## Comparison Chart

Use when the source contains tradeoffs, alternatives, old/new, before/after,
local/cloud, or cost/quality/speed comparisons.

Keep comparisons asymmetric when one option is intentionally preferred.
